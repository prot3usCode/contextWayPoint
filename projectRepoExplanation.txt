contextWayPoint Project Explanation
===================================

What this repository is
-----------------------
contextWayPoint is a lightweight compiler, router, and demo harness for
authored LLM context.

The core claim is:

    RAG retrieves relevant chunks.
    contextWayPoint defines and executes an authored route through context.

This repository is meant to show that many LLM failures are not just retrieval
failures. They are ordering failures. The model gets relevant facts, but it
gets them in the wrong order, mixed across workflows, or without a clear route.

What the packaged CLI does
--------------------------

The repository now includes an installable CLI called:

    contextwaypoint

The main commands are:

- validate
- compile
- route
- route-map
- fill-uuids
- demo

That means the project is no longer just a collection of standalone scripts.
It can now be installed in editable mode and run like a developer tool.


High-Level CLI Examples
-----------------------

Install locally:

    python -m pip install -e .

Validate all example files:

    contextwaypoint validate Formats

Compile a single global index:

    contextwaypoint compile Formats --out output/contextIndex.json

Render a routed packet:

    contextwaypoint route "Order Fulfillment Investigation" --mode step --format md

Render only the route map:

    contextwaypoint route-map "Order Fulfillment Investigation"

Fill blank problem UUIDs:

    contextwaypoint fill-uuids Formats --out filledFormats

Run the main project demo:

    contextwaypoint demo order-4-not-shipped


What the repository demonstrates
--------------------------------

At its best, the repository demonstrates this pattern:

    unordered retrieved context
               ->
    authored structured context
               ->
    compiled queryable index
               ->
    deterministic routed packet
               ->
    clearer, more traceable LLM input

The key idea is not just relevance.
The key idea is route.


Core Files and Folders
----------------------

README.md
  Quick explanation of the project, install steps, CLI commands, demo, and
  tests.

contextRouterWorkflow.md
  Practical runbook for install, validate, compile, route, fill UUIDs, demo,
  and tests.

pyproject.toml
  Package metadata for editable install and the CLI entrypoint.

build_backend.py
  Local lightweight build backend so the project can install cleanly in a more
  restricted environment.

src/contextwaypoint/
  Packaged implementation of the validator, compiler, router, demo flow, and
  CLI.

src/contextValidator.py
src/contextCompiler.py
src/contextRouter.py
src/queryContext.py
  Thin compatibility wrappers that still let the older script-style commands
  work.

Formats/
  Sample authored context files, including the multi-document order
  fulfillment example.

demos/rag_vs_contextwaypoint/
  Demo assets used to show the difference between unordered retrieval and an
  authored route.

docs/thesis.md
  Concise framing of the repository's thesis.

evals/
  Small evaluation scaffold with task definitions for comparing baseline,
  unordered, and routed prompts.

tests/
  Focused unit tests for core compiler and router behavior.


Current Core Features
---------------------

- Validate one file or a directory of YAML together
- Catch duplicate entry UUIDs across documents
- Catch duplicate problem UUIDs across documents
- Compile one file or many files into one flat JSON index
- Preserve hierarchy, path, depth, source order, source file, and source root
- Fill blank problem UUIDs to a new file, directory, or in place
- Route by step
- Route by weight
- Route by authored YAML order
- Route by keyword overlap
- Render packets as txt, md, or json
- Render route-only output
- Run a one-command demo
- Run a focused test suite


Current Demonstration Story
---------------------------

The clearest current story in the repository is:

1. Ask: Why has order 4 not shipped?
2. Show unordered retrieved context that is all relevant but badly staged.
3. Show the authored route through domain overview, order flow, payment,
   inventory, shipment, and explanation rules.
4. Generate the routed packet.
5. Make the difference obvious.

That is the point where the project starts to feel like a real tool rather
than an interesting script.


What is still missing
---------------------

- richer automated evaluation
- downstream model integrations
- a larger library of real workflow examples
- a GUI or authoring app layer
- packaging polish for broader distribution


Current milestone
-----------------

The current milestone is:

    A stranger can clone the repo, install it, run one command, and understand
    why ordered authored context is different from generic retrieval.
