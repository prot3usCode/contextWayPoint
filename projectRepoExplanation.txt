contextWayPoint Project Explanation
===================================

What this repository is
-----------------------
contextWayPoint is a lightweight context compiler and router for LLMs or agents.

The main idea is:

1. Author context as one YAML file or several related YAML files.
2. Tag sections with one or more problem flows.
3. Validate those files together.
4. Compile them into one flat JSON index.
5. Route only the entries needed for one problem.
6. Render the result as a source-aware packet or a route map.

The distinguishing claim is:

    RAG retrieves relevant chunks.
    contextWayPoint defines and executes an authored route.

What changed in the current code
--------------------------------

- Multi-document compile with `--input-dir`
- One global index with `source_file` and `source_root`
- Source-aware packet output
- `keyword` routing mode
- `--route-only` output
- Cross-document UUID validation
- A RAG comparison demo folder


Core Files
----------

README.md
  High-level overview and quick-start commands.

contextRouterWorkflow.md
  Practical runbook for validation, compile, routing, and demo usage.

src/contextValidator.py
  Validates one file or a directory of YAML files together.

src/contextCompiler.py
  Compiles one file or many files into a flat JSON index and can fill blank
  problem UUIDs for a single file.

src/contextRouter.py
  Thin entry-point script for the router.

src/queryContext.py
  Main routing logic. Matches entries by problem name, sorts by route mode,
  renders packets, and writes output files.

Formats/
  Sample authored context files, including a multi-document route example.

demos/rag_vs_contextwaypoint/
  Explanation folder contrasting unordered retrieval with an authored route.

Typical Commands
----------------

Validate all example files together:

    .venv/bin/python src/contextValidator.py --input-dir Formats

Compile them into one global index:

    .venv/bin/python src/contextCompiler.py --input-dir Formats

Render the cross-document packet:

    .venv/bin/python src/contextRouter.py "Order Fulfillment Investigation" --mode step --format md

Print only the route map:

    .venv/bin/python src/contextRouter.py "Order Fulfillment Investigation" --mode step --route-only --format txt

Use keyword mode:

    .venv/bin/python src/contextRouter.py "Order Fulfillment Investigation" --mode keyword --keywords payment failed shipment --format txt


What the repository demonstrates
--------------------------------

At its core, the repository demonstrates this pattern:

    broad documentation or retrieved chunks
               ->
    authored structured context
               ->
    compiled queryable index
               ->
    routed, traceable problem packet
               ->
    better-targeted LLM input
